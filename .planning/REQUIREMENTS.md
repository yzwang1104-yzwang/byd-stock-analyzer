## v1 Requirements

### 数据获取 (DATA)

- [ ] **DATA-01**: 通过 AkShare 获取比亚迪(002594)历史日线数据（OHLCV + 成交量），使用 qfq（前复权）价格
- [ ] **DATA-02**: 获取历史 PE/PB 估值数据，用于计算历史分位
- [ ] **DATA-03**: 获取行业 PE/PB 均值数据，用于同行对比
- [ ] **DATA-04**: 数据缓存到本地 CSV，仅在过期时重新获取；缓存数据需标注时间戳
- [ ] **DATA-05**: 数据获取接口接受任意股票代码参数（不为比亚迪硬编码，为 Phase 2 铺路）

### 技术分析 (TECH)

- [ ] **TECH-01**: 计算 SMA(20/50/200) 移动均线，判断中短期和长期趋势
- [ ] **TECH-02**: 计算 MACD(12/26/9)，生成金叉/死叉信号
- [ ] **TECH-03**: 计算 RSI(14)，判断超买(>70)/超卖(<30)状态
- [ ] **TECH-04**: 计算布林带(20,2)，判断价格波动区间和突破信号
- [ ] **TECH-05**: 计算 ATR(14)，辅助仓位计算和波动率判断
- [ ] **TECH-06**: 计算成交量 SMA(20)，确认价格变动的量能支持
- [ ] **TECH-07**: 所有指标使用 `shift(1)` 防止前瞻偏差——信号日 T 只使用 T-1 及之前的数据

### 估值分析 (VALU)

- [ ] **VALU-01**: 计算当前 PE 在历史 PE 数据中的百分位（用 scipy.stats.percentileofscore）
- [ ] **VALU-02**: 计算当前 PB 在历史 PB 数据中的百分位
- [ ] **VALU-03**: 与行业 PE/PB 均值对比，输出相对估值判断（低估/合理/高估）
- [ ] **VALU-04**: 估值模块在 AkShare 数据不可用时优雅降级——给出降级提示但不阻断整体评分

### 评分与信号 (SCOR)

- [ ] **SCOR-01**: 加权综合评分模型——估值(35%) + 技术(30%) + 趋势(20%) + 成交量(10%) + 情绪(5%)
- [ ] **SCOR-02**: 评分输出为整数(0-100)，带置信度标注（高/中/低）
- [ ] **SCOR-03**: 操作建议映射——0-30=卖出/回避, 31-55=观望等待, 56-75=可考虑买入, 76-90=建议买入, 91-100=强烈买入
- [ ] **SCOR-04**: 仓位建议四档——25%/50%/75%/100%，基于评分和 ATR 波动率
- [ ] **SCOR-05**: 评分权重可配置（通过 config.py），不硬编码
- [ ] **SCOR-06**: 包含看跌因子以防止确认偏差——模型必须能在看跌条件下给出卖出信号
- [ ] **SCOR-07**: 信号冷却机制——连续两次买入信号之间至少间隔 N 个交易日

### 卖出信号 (SELL)

- [ ] **SELL-01**: 基础卖出触发——RSI > 70 且 PE > 80% 历史分位 → 卖出信号
- [ ] **SELL-02**: 卖出信号也输出评分、仓位建议（降至 0-25%）和关键依据

### 输出与体验 (OUT)

- [ ] **OUT-01**: 主输出为一行结论——评分 + 操作建议 + 仓位 + 一句依据（如"评分 78/100，建议买入 50% 仓位——RSI 超卖 + PE 低于历史 70% 分位"）
- [ ] **OUT-02**: 使用 Rich 库格式化终端输出——彩色面板 + 结构化表格
- [ ] **OUT-03**: 详情展开模式——用户可选查看每个指标的详细数值
- [ ] **OUT-04**: 命令行参数支持——`--price` 指定当前股价，`--verbose` 查看详情
- [ ] **OUT-05**: 输出包含信号时效性提示——"基于今日收盘数据，建议在 24 小时内决策"

### 合规 (COMP)

- [ ] **COMP-01**: 所有输出底部强制标注免责声明："分析结果仅供参考，不构成任何投资建议。投资有风险，入市需谨慎。"
- [ ] **COMP-02**: 不使用命令式措辞——用"综合评分 78/100，历史类似评分后上涨概率约 65%"而非"建议买入"
- [ ] **COMP-03**: 不给出具体目标价位，不承诺收益，不使用"保证""预计收益"等措辞

## v2 Requirements（Phase 2 推迟）

- [ ] **v2-01**: 信号历史存储和追踪——"最近 5 次买入信号：3 次正确，平均涨幅 8%"
- [ ] **v2-02**: 蜡烛图形态识别——锤子线/吞没形态/十字星
- [ ] **v2-03**: 背离检测——价格创新低但 RSI 不创新低
- [ ] **v2-04**: 多时间框架对齐——日线/周线/月线信号一致性
- [ ] **v2-05**: Django Web 仪表盘 + ECharts K 线图
- [ ] **v2-06**: 每日邮件推送评分变化
- [ ] **v2-07**: 港股比亚迪(1211.HK)支持

## Out of Scope（明确不做）

- 多股票筛选器 — Phase 1 只做比亚迪，单股票深度 > 多股票浅覆盖
- 机器学习/LSTM 预测 — 违反可解释性要求
- 新闻情感分析 — 用户表示"新闻真假难辨"，反而引入噪音
- 实时盘中推送 — Phase 1 是命令行脚本
- 回测引擎 — Phase 2 用信号追踪替代
- 用户自定义指标参数 — 用户不会调参
- 期权/期货分析 — 用户只做股票现货
