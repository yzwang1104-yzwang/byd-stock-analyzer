# FEATURES.md — 比亚迪股票分析工具功能生态

## Research Summary

**Project:** BYD Stock Analyzer
**Mode:** Ecosystem — Features dimension
**Confidence:** MEDIUM

## Core Insight

**主要差异化 = "结论优先架构"** — 现有所有工具（同花顺、雪球、TradingView、东方财富）都是给用户数据让用户自己分析。这个工具替用户做分析，给出结论。用户已验证："我会跟着提示直接建仓。"

---

## Table Stakes（必须有）

| 功能 | 复杂度 | 依赖 | 说明 |
|------|--------|------|------|
| 价格数据获取 | Low | AkShare | 比亚迪历史日线数据+实时行情 |
| 技术指标计算 | Medium | TA-Lib | MACD, RSI, MA(5/10/20/60), 布林带, 成交量 |
| 估值指标 | Medium | AkShare | PE/PB 历史分位、行业对比 |
| 综合评分模型 | High | 以上全部 | 加权计算买入评分 (0-100) |
| 买卖信号输出 | Low | 评分模型 | 买入/卖出/观望 + 置信度 |
| 仓位建议 | Low | 评分模型 | 基于分数和风险偏好的仓位百分比 |
| 关键依据说明 | Medium | 评分模型 | 1-2句话解释为什么是这个评分（可解释性） |

---

## Differentiators（差异化优势）

| 功能 | 复杂度 | Phase | 说明 |
|------|--------|-------|------|
| **结论优先输出** | Low | 1 | "现在买入评分 85 分，建议 30% 仓位"——一句话，不是一屏数据 |
| **单股票深度分析** | Medium | 1 | 所有指标只为比亚迪优化，不做通用稀释 |
| **信号历史追踪** | Medium | 2 | 存储每次信号+后续实际走势，证明工具可信度 |
| **可解释评分** | Medium | 1 | "RSI超卖 + MACD底背离 + PE低于历史70%分位" |
| **多时间框架对齐** | High | 2 | 日线/周线/月线信号一致性检查 |
| **蜡烛图形态识别** | Medium | 2 | 锤子线/吞没形态/十字星等关键形态 |
| **背离检测** | Medium | 2 | 价格创新低但 RSI 不创新低 = 底背离 |

---

## Anti-Features（故意不做）

| 不做 | 原因 |
|------|------|
| 多股票筛选器 | 单股票深度 > 多股票浅覆盖，Phase 1 只做比亚迪 |
| 社交/社区功能 | 与核心价值无关，增加复杂度和合规风险 |
| 实时盘中推送 | Phase 1 是命令行脚本，没有实时需求 |
| 机器学习黑盒预测 | 用户要求可解释，黑盒 AI 违背核心价值 |
| 技术指标自定义参数 | 用户不会调参数，默认最优配置即可 |
| 期权/期货分析 | 用户只做股票现货 |
| 投资组合管理 | Phase 1 只关注单一标的 |
| 新闻情感分析 | 用户表示"新闻不确定真假"，反而引入噪音 |
| 回测引擎 | Phase 2 再考虑，Phase 1 用信号追踪代替 |

---

## Phase 1 最小可行链

```
AkShare获取数据 → TA-Lib计算指标 → PE/PB分位 → 综合评分 → 买卖信号+仓位+依据
```

这条链跑通了，整个产品就验证了。

## Phase 2 扩展

```
+ 信号历史存储和追踪
+ 蜡烛图形态 + 背离检测
+ 多时间框架对齐
+ 简单的 HTML 仪表盘
```
